# Auteur v0.8.1 — Packaging Fix for `data/composition/`

## Root cause

`CompositionRulesLoader` in `src/auteur/narrative_orchestration/schema/rules_loader.py`
resolved its YAML resource using:

```python
Path(__file__).parent.parent.parent.parent.parent / "data" / "composition" / "composition_constraints.yaml"
```

This traversal worked from a source checkout where `data/` sits at the project
root, but failed from an installed wheel where `__file__` resolves to the
`site-packages` directory and `data/composition/` was not included in the wheel.

## Why source/editable installs masked it

- **Source checkout**: the `data/` directory exists at the repository root,
  so `Path(__file__).parents[5]` reaches it correctly.
- **Editable install** (`pip install -e .`): `__file__` points back to the
  source checkout, so the traversal works despite the wheel not containing
  `data/composition/`.
- **Wheel install**: `__file__` points to `site-packages/auteur/...` where
  the `data/` directory does not exist — exposing the defect.

## Fix

### 1. Moved data under package-owned path

```
Before:  data/composition/composition_constraints.yaml    (project root)
After:   src/auteur/data/composition/composition_constraints.yaml
Before:  data/composition/ownership_rules.yaml             (project root)
After:   src/auteur/data/composition/ownership_rules.yaml
```

Added `src/auteur/data/composition/__init__.py` making it a proper Python
package (`auteur.data.composition`).

### 2. Updated loader to use package-resource APIs

```python
# Old (brittle path traversal):
yaml_path = Path(__file__).parent.parent.parent.parent.parent / "data" / "composition" / "composition_constraints.yaml"

# New (package resources):
import importlib.resources as resources
ref = resources.files("auteur.data.composition").joinpath("composition_constraints.yaml")
```

The `yaml_path` parameter on `CompositionRulesLoader.__init__()` is preserved
for backward compatibility. When `None` (default), the new `importlib.resources`
path is used.

### 3. Packaging (no changes needed)

`pyproject.toml` already has `packages = ["src/auteur"]` which automatically
includes everything under `src/auteur/`. The new `src/auteur/data/composition/`
files are included without additional configuration.
