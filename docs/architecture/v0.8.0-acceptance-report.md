### Pre-existing failure detail

All 25 failures are from a single file:
`tests/auteur/narrative_orchestration/test_rules_loader.py`

Cause: The `CompositionRulesLoader` uses a hardcoded path relative to
`__file__` to locate `data/composition/composition_constraints.yaml`.
This path resolves to the installed package's `site-packages` directory,
but the `data/` directory (at project root, not under `src/auteur/`) is
not included in the wheel's `package-data`. This environment issue is
not caused by v0.8.0 changes â€” confirmed pre-existing via `git stash`.
