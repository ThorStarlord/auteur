# Auteur v0.9.0 — Author Review Sessions

## Overview

v0.9.0 turns the v0.8 decision-orchestration capabilities into a coherent
author-facing review workflow. It introduces review sessions that guide an
author through the highest-priority open decision, evidence inspection,
choice resolution, acceptance preparation, authority-bearing acceptance,
and downstream impact refresh.

## Architecture

### Package structure

```
src/auteur/review/
    __init__.py          — exports
    models.py            — typed session, event, choice, state models
    persistence.py       — immutable event storage, session pointers
    service.py           — application-service boundary
    selection.py         — deterministic review-target selection
    cli.py               — argparse subcommands and handlers
```

### Reused existing systems (NOT duplicated)

| Capability | Reuses |
|-----------|--------|
| Decision discovery | `DecisionWorkspaceService.list_decisions()` |
| Decision detail | `DecisionWorkspaceService.inspect()` |
| Readiness | `DecisionReadiness` enum |
| Conflicts | `ConflictDetector` / `DecisionWorkspaceService.conflicts()` |
| Candidate comparison | `DecisionWorkspaceService.compare_candidates()` |
| Acceptance prep | `DecisionWorkspaceService.prepare_acceptance()` |
| Acceptance execution | Existing expression/acceptance subsystem |
| Impact preview | `DecisionWorkspaceService.impact_preview()` |
| Workflow actions | `WorkflowEngine` with decision service |
| Authority model | `AuthorityLevel` from workflow models |

## Session model

### Lifecycle states

```
OPEN → INSPECTING → AWAITING_CHOICE → READY → PREPARED →
  AWAITING_ACCEPTANCE → ACCEPTING → ACCEPTED → REFRESHING → COMPLETED
```

Failure branches from PREPARED, ACCEPTING, REFRESHING return to the
prior state with error diagnostics preserved. Staleness detection can
mark any active state as STALE.

### Events (append-only)

- `SESSION_STARTED`
- `TARGET_SELECTED`
- `EVIDENCE_SNAPSHOTTED`
- `CHOICE_RECORDED`
- `CHOICE_SUPERSEDED`
- `PREPARATION_BLOCKED`
- `PREPARATION_COMPLETED`
- `ACCEPTANCE_REQUESTED`
- `ACCEPTANCE_REFUSED`
- `ACCEPTANCE_COMPLETED`
- `IMPACT_REFRESH_STARTED`
- `IMPACT_REFRESH_FAILED`
- `IMPACT_REFRESH_COMPLETED`
- `SESSION_MARKED_STALE`
- `SESSION_RESUMED`
- `SESSION_COMPLETED`
- `SESSION_ABORTED`

Each event carries: event_id, session_id, sequence, event_type, timestamp,
authority, payload dict, source_refs dict, previous_event_hash, event_hash.

## Integration boundaries

### Review → Decision Workspace

Review sessions reference decisions by ID. The decision workspace provides
readiness, evidence, candidates, conflicts, and next-action recommendations.
Review never modifies decisions directly.

### Review → Acceptance

Acceptance preparation reuses `DecisionWorkspaceService.prepare_acceptance()`.
Actual acceptance is delegated to the existing acceptance subsystem.
Review code never writes canonical pointers directly.

### Review → Impact

Impact preview and refresh reuses `DecisionWorkspaceService.impact_preview()`.
The service's `refresh_snapshots()` and `refresh()` are used for post-acceptance
workspace updates.

### Review → Workflow

Workflow engine integrates review sessions through the existing decision-aware
`recommend_actions()`. Review sessions add new action types for start/resume/
choose/accept that flow through the existing authority and recursion protections.

## CLI

```
auteur review start [--decision <id>]    — start or resume review session
auteur review status                     — show current review session
auteur review inspect <session-id>        — full session detail
auteur review choose <session-id>         — resolve an author choice
  --choice <id> --option <value> [--reason <text>]
auteur review prepare <session-id>        — prepare acceptance
  --candidate <id>
auteur review accept <session-id>         — perform acceptance
  --candidate <id> --confirm
auteur review resume <session-id>         — resume after interruption
auteur review history <session-id>        — event history
auteur review list                        — list all sessions
auteur review abort <session-id>          — abort active session
```

All commands support `--project` and `--json`.

## Persistence

```
.auteur/
  reviews/
    sessions/        — session state snapshots (immutable)
    events/          — append-only event chain per session
    latest/          — atomic pointer to current session
```
