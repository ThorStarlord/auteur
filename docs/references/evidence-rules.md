# Evidence Rules for Repository Sensemaking

All claims in a repository sensemaking brief must be evidence-backed.

1. File Citation: Name the exact file and line references when possible.
2. Structural Proof: Use directory layout and file organization as evidence for current shape.
3. Contrastive Evidence: Compare docs claims against code/tree output.
4. Logic Trace: Follow actual execution paths for workflow/contract assertions.
5. No Vibe-Based Diagnosis: Do not claim uncertainty without concrete proof of weakness.
