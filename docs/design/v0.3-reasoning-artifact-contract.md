# v0.3 Reasoning-Run Artifact Contract

## Artifact family

Three artifact types, all `authority: derived`, `canonical: false`:

### 1. ReasoningRunRecord
- **Path**: `chapters/<NN>/reasoning/runs/reasoning_run_<chapter>_<iteration>_<id>.yaml`
- **Lifecycle**: immutable after write
- **Fields**: run_id, chapter_index, iteration, plan_id, source_hashes (draft, outline, blueprint, bible), critic_ids, runtime_version, created_at, overall_status, outcome_refs, review_ref

### 2. CriticOutcomeReport
- **Path**: `chapters/<NN>/reasoning/outcomes/<run_id>/<critic_id>.json`
- **Lifecycle**: immutable after write
- **Fields**: report_id, critic_id, critic_version, plan_id, status (success/failed/stale/blocked), findings (list), source_snapshot, error (optional)

### 3. SynthesizedReview
- **Path**: `chapters/<NN>/reasoning/reviews/reasoning_review_<run_id>.yaml` + `.md`
- **Lifecycle**: immutable after write
- **Fields**: review_id, run_id, overall_status, blocking_findings, warnings, synthesis, recommended_actions, freshness

### Mutable pointer
- **Path**: `chapters/<NN>/reasoning/latest.yaml`
- **Lifecycle**: overwritten each new run
- **Fields**: run_id, review_ref, created_at

## Freshness
- Source hashes recorded at run time
- `report_is_fresh()` compares current hashes against recorded
- Staleness detected when draft, outline, blueprint, or bible changed
- Historical runs never updated

## Retention
- No automatic cleanup
- Previous runs remain inspectable in `runs/` and `outcomes/`
- Only `latest.yaml` is mutable
