## 2. Architecture Summary

The convergence subsystem lives in `src/auteur/convergence/` with 9 modules:

| Module | Responsibility |
|---|---|
| `models.py` | Typed Pydantic models for targets, obligations, preservation, candidates, comparison, proposals, state |
| `scope.py` | Deterministic target resolution from scope parameters and impact findings |
| `obligations.py` | Obligation collection from story_identity, blueprint, outline, scene, impact sources |
| `preservation.py` | Preservation analysis using structural boundaries and beat IDs |
| `candidates.py` | Candidate lifecycle (generate, register, list, update status), CandidateStore |
| `comparison.py` | Deterministic multidimensional comparison over evidence |
| `planner.py` | Reconciliation proposal creation with conflict detection and authority-required choices |
| `persistence.py` | Immutable convergence artifact storage with atomic pointer management |
| `cli.py` | CLI subcommand registration and handler dispatch |

### Integration points

- **Impact (v0.5.0):** `resolve_target_from_impact()` maps impact findings to convergence targets
- **Workflow (v0.4.0):** Workflow rules recommend `auteur realization status` as a REALIZATION stage action
- **Provenance:** Content hashing patterns follow `sha256:` prefix convention
- **Acceptance:** No parallel acceptance; all candidates remain noncanonical

## 6. Limitations

1. **Candidate generation** creates metadata but not actual prose content (content_artifact_ref is empty for generated candidates). Full generation requires the expression pipeline.
2. **Reasoning integration** tracks evaluation references but does not call the reasoning runtime directly — that requires a model provider.
3. **Chapter-level acceptance** is not implemented in convergence — it uses the existing acceptance API.
4. **Workflow integration** is at the action-recommendation level, not fully consumed by the workflow engine.

## 7. Deferred Scope

| Feature | Reason |
|---|---|
| LLM-backed candidate generation | Requires model provider; CLI generates metadata only |
| Automatic freshness recomputation | Deferred to minimize coupling |
| Full workflow engine consumption | Existing workflow engine is action-based; deeper integration deferred |
| Beat-range CLI shorthand | Scene-level is the primary use case |
