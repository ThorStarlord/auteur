## Executive Summary

v0.7.0 completes the Author Decision Workspace as a genuine vertical integration across Auteur's existing subsystems. The release composes real project state from impact findings, convergence candidates, reasoning evidence, and artifact provenance into typed decision models. All seven CLI commands now perform meaningful operations against actual project data.

The implementation preserves existing design patterns (immutable snapshots, deterministic IDs, frozen dataclasses), respects subsystem boundaries (read-only composition, no canonical mutation), and includes 41 comprehensive tests proving both individual models and integrated workflows.

## Implementation Scope

### 1. Service Layer ✓ COMPLETE

**File:** `src/auteur/decision/service.py` (new)

`DecisionWorkspaceService` orchestrates:
- Real project root discovery and validation
- Integration with ArtifactStore (provenance metadata)
- Integration with ImpactStore (structural revision findings)
- Integration with ConvergenceStore (targets, candidates, comparisons, proposals)
- Composition of typed AuthorDecision models
- Deterministic readiness evaluation
- Acceptance preparation with multi-checkpoint validation

Key methods:
- `status()` — workspace overview
- `list_decisions(chapter_index=None)` — filtered decision enumeration
- `inspect(decision_id)` — full decision detail with evidence
- `evidence_for_decision(decision_id)` — evidence grouped by classification
- `compare_candidates(decision_id)` — candidate comparison data
- `next_action(decision_id)` — deterministic next-action recommendation
- `prepare_acceptance(decision_id, candidate_id)` — read-only readiness validation
- `refresh()` — reload state from subsystems

### 3. Real Subsystem Integration ✓ COMPLETE

#### Impact Integration
- Loads impact findings from `ImpactAnalyzer` or persisted reports
- Maps findings to decisions with stable artifact references
- Preserves severity and dependency-path evidence
- Creates one decision per unresolved impact target

#### Convergence Integration
- Loads revision targets from `ConvergenceStore`
- Loads candidates with status, freshness, obligations, preserved regions
- Loads comparisons with dimensions and conflicts
- Loads reconciliation proposals with unresolved obligations
- Computes multidimensional candidate summaries

#### Provenance Integration
- Uses `ArtifactStore` for artifact metadata, freshness, and dependencies
- Preserves content hashes and revision tracking
- Respects lifecycle (draft/proposed/accepted/replaced)
- Validates knowledge coherence for scene realizations

#### Artifact Store Integration
- Validates project root (.auteur directory)
- Loads artifact metadata for provenance tracking
- Queries current and revision history

### 4. Decision Identity ✓ COMPLETE

Stable deterministic IDs computed from:
- Project name
- Trigger type (impact_finding, convergence_target, etc.)
- Chapter index
- Scene ID
- Sorted beat IDs
- Target artifact ID
- Sorted trigger IDs

Same structured state → same decision ID. Equivalent unresolved state resolves to same decision identity. 16-character SHA256 digest.

### 5. Readiness Evaluation ✓ COMPLETE

Explicit evaluation order with no ambiguity:

```
1. Malformed/missing provenance → BLOCKED
2. Stale target or evidence → STALE
3. No candidate → NEEDS_CANDIDATE
4. Candidate lacks required evaluation → NEEDS_EVALUATION
5. Multiple candidates lack comparison → NEEDS_COMPARISON
6. Reconciliation required → NEEDS_RECONCILIATION
7. Author choice unresolved → NEEDS_AUTHOR_DECISION
8. All prerequisites satisfied → READY_FOR_ACCEPTANCE
9. Already accepted → RESOLVED
```

Documented readiness transitions with focused tests for each state.

### 6. Evidence Classification ✓ COMPLETE

Evidence grouped by classification:
- **FACT** — Structural facts, current observation
- **DERIVED_INFERENCE** — Impact findings, stale dependencies, conflicts
- **RECOMMENDATION** — Suggested next actions
- **AUTHOR_CHOICE** — Unresolved author decisions

Freshness tracked per evidence item (current/stale/unknown).

### 7. Acceptance Preparation ✓ COMPLETE

`prepare_acceptance()` validates without performing acceptance:
1. Candidate exists in decision
2. Candidate freshness is current
3. Reasoning evidence present
4. Obligations satisfied (none unsatisfied)
5. Unresolved author choices absent
6. Reconciliation conflicts resolved

Returns `AcceptancePreparation` with:
- `is_ready` (boolean)
- `blockers` (list of reasons)
- `verification_results` (dict of checkpoint statuses)
- `affected_downstream` (predicted impact)

### 8. Persistence ✓ COMPLETE

Immutable decision snapshot storage:
- `.auteur/decisions/snapshots/{decision_id}.json` — metadata snapshots
- `.auteur/decisions/latest/latest.json` — global latest pointer
- `.auteur/decisions/latest/latest-by-target-{artifact_id}.json` — target-specific pointers

Atomic writes with conflict detection:
- Same ID + identical state → idempotent success
- Same ID + different state → explicit conflict error
- Stale snapshots retained for history

## Known Limitations

### Intentionally Deferred

- **Workflow integration:** next_action() recommends actions but does not execute them or recursively dispatch workflow commands
- **Automated acceptance:** prepare_acceptance() validates but does not perform acceptance
- **Reconciliation module:** Not implemented; conflicts loaded from convergence proposals only
- **Acceptance module:** Not implemented; acceptance remains outside decision workspace scope

### Design Constraints Preserved

- **No canonical mutation:** Service never updates canonical pointers
- **Read-only subsystem composition:** All subsystem APIs used in read mode only
- **Immutable decisions:** Once persisted, snapshots are immutable
- **Deterministic assembly:** Same state always produces same decision ID and readiness

## Summary

The v0.7.0 Author Decision Workspace completes as a genuine, trustworthy integration of Auteur's impact, convergence, and artifact subsystems. The release demonstrates that the decision pattern generalizes across multiple decision sources, that real subsystem composition is achievable without special cases, and that author-facing decision support can exist without claiming narrative authority.

Every advertised CLI command now performs its operation against real project state. Tests prove subsystem integration, decision model correctness, and boundary preservation. The architecture remains open for future extensions (additional decision triggers, reconciliation automation, workflow integration) without infrastructure changes.
